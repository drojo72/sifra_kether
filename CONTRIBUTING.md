# Contributing to Sifra Kether

Thank you for your interest in contributing to Sifra Kether. This project is part of the KeySign Labs ecosystem and follows an editorial-first philosophy.

## How to Contribute

1. **Fork the relevant repository** (api, web, mobile, or branding)
2. **Create a feature branch** from `main`
3. **Follow the coding standards** of the specific sub-project
4. **Submit a Pull Request** with clear description

## Areas of Contribution

- **Textual metadata**: Canonical references, translations, manuscript variants
- **Editorial structures**: Study schemas, commentary hierarchies, publication workflows
- **Manuscript references**: OCR pipelines, paleographic annotations, provenance data
- **API improvements**: Endpoints, documentation, performance
- **Visual assets**: Brand-compliant illustrations, diagrams, editorial layouts
- **Comparative theology**: Parallel texts, thematic links, scholarly apparatus

## Standards

- All contributions must align with the project's textual, editorial, and research standards
- Code must pass linting and tests defined in the sub-project
- Database changes must include migration scripts
- API changes must include documentation updates

## Review Process

All contributions are reviewed by maintainers. Editorial and theological contributions may additionally be reviewed by the project's scholarly advisory panel.

## Code of Conduct

Be respectful, constructive, and focused on the quality of the knowledge being built.

## Questions?

Open an issue in the relevant repository or contact the maintainers.
